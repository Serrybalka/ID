# Прототип сервиса подписания исполнительных документов

Локальный desktop-прототип раздела «Исполнительные документы» Единой платформы «Цифровое правосудие». Интерфейс предназначен для демонстрации заказчику и юзабилити-тестирования сценариев просмотра, фильтрации, выбора, подписания и контроля отправки исполнительных документов в ФССП.

## Стек

- React 18
- TypeScript
- Vite
- React Router
- Lucide React
- Vitest
- React Testing Library
- Локальные mock-данные
- localStorage для настроек колонок, плотности, меню и пагинации

## Структура

```text
src/
  app/                         маршрутизация приложения
  components/
    common/                    badge, modal, toast
    document-details/          выезжающая карточка ИД
    filters/                   модальное окно фильтров
    layout/                    header и sidebar
    registry/                  toolbar, tabs, таблица, пагинация, настройки
    signing/                   пакетное подписание
  data/                        mock-реестр документов
  hooks/                       localStorage hook
  pages/                       страница раздела
  types/                       строгие TypeScript-типы
  utils/                       фильтрация, форматирование, правила подписания
```

## Команды

```bash
npm install
npm run dev
npm run build
npm run test
npm run lint
```

## Реализованные сценарии

- Реестр из 96 разных исполнительных документов.
- Вкладки по статусам с пересчетом количества.
- Быстрый поиск по делу, номеру ИД и должнику.
- Расширенные фильтры в модальном окне, chips активных фильтров.
- Таблица со sticky header, горизонтальной прокруткой, сортировкой, фиксированной первой и последней колонкой.
- Единичный и пакетный выбор только документов, доступных для подписания.
- Сохранение видимости колонок, плотности таблицы, ширины меню и размера страницы.
- Выезжающая справа карточка документа с вкладками: общие сведения, история отправки, постановления СПИ, файлы, ошибки, журнал действий.
- Переключение между строками без закрытия карточки.
- Развернутый режим карточки на всю рабочую область.
- Mock-сценарий пакетного подписания УКЭП с прогрессом и частичным успехом.
- Повторная отправка, скачивание, печать, проверка подписи через toast-уведомления.
- Базовая доступность: фокус, aria-атрибуты, dialog role, tablist, alert, Escape для модальных окон и карточки.

## Mock-логика

Данные генерируются локально в `src/data/documents.ts`: различаются статусы, суммы взыскания, ФИО должников, номера дел, номера ИД, форматы ЭП, наличие постановлений СПИ, ошибки отправки, файлы, история и аудит. Операции подписания и обновления имитируют задержку, чтобы показать loading/progress состояния.

## Ограничения прототипа

- Нет серверной части и реальной интеграции с ФССП или удостоверяющим центром.
- Изменение ширины колонок и drag-and-drop порядка колонок представлены настройкой видимости; полноценный resize/drag можно добавить отдельным слоем.
- Файлы не скачиваются физически, операции подтверждаются toast-уведомлениями.
